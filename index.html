Formation Git & GitHub

Git est un système de contrôle de version distribué.

  • Suivre les modifications des fichiers
  • Revenir à des versions précédentes
  • Permettre à plusieurs personnes de travailler sur le même projet
  • Conserver l'historique du code

Installation

  • Télécharger Git
  • git -v : Vérifier l'installation

Configuration

  • git config --global user.name "alex"
  • git config --global user.email "alex@gmail.com"
  • git config --list
  • git --help : Aide Git

Initialiser un dépôt Git

git init : Crée un dossier caché .git

Les zones Git

  • Working Directory (Zone de travail)
  • Staging Area (Zone d'index)
  • Repository (Zone de dépôt)

Les commandes Git

  • git status : Affiche l’état actuel du dépôt
  • git add
    • git add nom_fichier
    • git add .
  • git reset : Retirer un fichier de l’index
  • git commit -m "message" : Valider les modifications
  • git log
    • --oneline
    • -p
  • git show : Voir un commit en détail
  • git diff
    • Voir les modifications non indexées
    • --cached

Les pointeurs

  • main : Branche principale
  • HEAD : Pointe vers la branche actuelle
  • Tag : Référence fixe sur un commit

Créer un tag

  1. git tag v1.0
  2. git tag -d v1.0
  3. git push origin v1.0

Branches

Une branche est une version parallèle du code permettant de développer une fonctionnalité sans modifier la branche principale.

  • git branch
  • git branch nom_branche
  • git switch nom_branche
  • git branch -d nom_branche

Fusion (Merge)

  • git switch main
  • git merge nom_branche

GitHub

GitHub est une plateforme d’hébergement de code basée sur Git.

  • Inscription : https://github.com/
  • git clone https://github.com/utilisateur/projet.git
  • Gérer les remotes :
    • git remote -v
    • git remote add origin https://github.com/utilisateur/projet.git
  • Envoyer les modifications :
    • git push -u origin main
    • git push origin --tags
  • Récupérer les modifications :
    • git pull
    • git fetch

Travail en équipe

git blame nom_fichier : Voir qui a modifié chaque ligne.

gitk : Interface graphique Git

GitHub avancé

GitHub propose des outils collaboratifs puissants pour gérer un projet au-delà du simple contrôle de version.

Pull Request (PR)

Une Pull Request permet de proposer des modifications d’une branche vers une autre (souvent vers main) et d’en discuter avant de fusionner.

  • Créer une branche : git checkout -b nouvelle-feature
  • Commiter les modifications : git add . puis git commit -m "Ajout feature"
  • Pusher la branche : git push origin nouvelle-feature
  • Sur GitHub : Créer la Pull Request pour revue et fusion

Fork

Forker un dépôt crée une copie personnelle du projet sur votre compte GitHub, ce qui permet de contribuer sans toucher au dépôt original.

  • Cliquez sur Fork sur la page GitHub du projet
  • Clonez votre fork : git clone https://github.com/votre-compte/projet.git
  • Créez vos branches et PR pour proposer vos changements

Issues

Les Issues servent à suivre les bugs, idées et tâches à faire dans un projet.

  • Créer une Issue pour signaler un bug ou proposer une amélioration
  • Assigner des labels, des milestones et des personnes responsables
  • Commenter et suivre l’avancement jusqu’à la résolution

Actions

GitHub Actions permet d’automatiser des tâches comme tests, déploiements, ou intégration continue.

  • Créer un fichier .github/workflows/main.yml
  • Définir les jobs et steps : build, test, déploiement
  • Chaque push ou Pull Request peut déclencher automatiquement ces workflows
